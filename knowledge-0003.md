## 1. 記事タイトル
GitHub Copilotでサブエージェントたちに会話をさせる
## 2. 公開日・更新日
2026年02月04日
## 3. 内容要約
・全体を貫くメインテーマ
GitHub Copilotのカスタムエージェント（サブエージェント）機能を利用し、異なる専門性や役割（人格）を持つ複数のAI同士を会話させ、人間が介入する前の情報整理やディスカッションを自動化する試みの検証。
・具体的な重要ポイント
 * 複数エージェントの設定： フロントエンド担当（侍風）と運用担当（関西商人風）など、性格や役割が異なるカスタムエージェントを用意。
 * GitHub Copilot CLIの活用： 本検証時点ではVS Code上のエージェントモードよりもCLIの方がモデルの明示的な指定や詳細なログ確認がしやすいため、CLIベースで実行。
 * モデルの使い分け： 親エージェントに「Claude 3.5 Opus」などの高度なモデルを配置し、サブエージェントを制御・ハンドリングさせることで、安定した対話を実現。
 * 現時点での制約： VS Code上ではサブエージェントが親のモデルを継承してしまう場合があることや、プロンプトで明示しないと指定モデルが使われない等の課題がある。
・筆者の結論や読者へのアドバイス
サブエージェント同士に会話させることで、人間がチェックする前にある程度の要件定義や懸念点の洗い出しを完了させることが可能です。今回は会話をさせる実験でしたが、今後は会話ログのファイル出力や成果物の共同作成など、より実戦的な自動化フローへの応用が期待されます。
